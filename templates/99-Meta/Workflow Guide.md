# Obsidian Workflow Guide

## The Inbox System 📥

**Rule #1: When in doubt, dump it in the Inbox**

### Daily Routine (5 minutes)
1. **Morning:** Create today's daily note from template
2. **Throughout the day:** Capture everything in Inbox or daily note
3. **Evening:** Quick review - move 2-3 items from Inbox to proper locations

### Weekly Review (15 minutes)
1. **Clean out Inbox:** Sort everything into proper folders
2. **Review daily notes:** Extract important items to permanent notes
3. **Update project status:** Check what's active vs. archived

## Quick Capture Hotkeys
- `Cmd+N` → New note (put in Inbox)
- `Cmd+T` → Today's daily note
- `Cmd+Shift+N` → New note from template

## Naming Quick Reference
- **Daily notes:** `YYYY-MM-DD Daily Note.md`
- **Meetings:** `YYYY-MM-DD Meeting Name.md`
- **Everything else:** `Title Case Description.md`
- **See:** [[Naming Conventions Guide]] for full details

## Folder Guidelines

### 00-Inbox
- Everything starts here
- No subfolders needed
- Review weekly

### 01-Daily  
- One note per day: `YYYY-MM-DD Daily Note`
- Meeting notes: `YYYY-MM-DD Meeting Name`
- Quick format: Date in filename, consistent structure

### Alex/Learning
- Technical courses and tutorials
- Use status tags: #learning #practicing #mastered
- Include practical examples you can copy

### Alex/Projects
- Personal development projects
- Include tech stack and progress logs
- Link related learning notes

### Alex/Resources
- Reading lists and reference materials
- Bookmarked articles and tools
- Quick lookup information

### Datadog/Active
- Current work projects only
- Move to Archive when done
- Include status and next steps

### Datadog/Scripts
- Keep your existing structure - it works!
- Add brief descriptions at the top of each note

### Datadog/Knowledge
- Best practices and lessons learned
- Process documentation
- Things you want to remember for future courses

### Datadog/Templates
- Course templates
- Standard structures
- Reusable components

## Tags Strategy
Keep it simple - only use tags that you'll actually search for:
- `#meeting` - meeting notes
- `#idea` - random ideas to develop
- `#todo` - action items
- `#learning` - educational content
- `#resource` - reference materials
- `#active` - current projects
- `#review` - needs follow-up

## Links Strategy
- Use `[[]]` for connecting related ideas
- Link projects to learning notes
- Connect meeting notes to project notes
- Build your personal knowledge graph naturally
